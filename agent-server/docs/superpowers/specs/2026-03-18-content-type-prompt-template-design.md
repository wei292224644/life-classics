# content_type_rules.json — prompt_template 设计规范

**日期**：2026-03-18
**文件**：`agent-server/app/core/parser_workflow/rules/content_type_rules.json`
**目标**：为 RAG 向量检索优化 transform_node 的 LLM 转化提示词

---

## 背景

`transform_node` 将分类后的文档片段（segment）通过 LLM 转化为向量化文本，存入 ChromaDB。当前所有 semantic_type 的 `prompt_template` 均为同一占位符，未针对内容特征做差异化设计。

**prompt 构造方式**（transform_node.py）：

```python
prompt = f"""
按照以下提示词，处理原文本。
{transform_params["prompt_template"]}

原文本：
{content}
"""
```

`prompt_template` 直接拼入 LLM 调用，是针对该类型内容的完整转化指令。

---

## 设计目标

- **主要目标**：提高 RAG 向量检索的召回率和准确率
- **扩展策略**：适度扩展（允许补充语义注释），但原始数值、限量符号、单位等精确内容绝对不得改动

---

## 真实数据特征分析

基于 GB 15044—2016（食品添加剂 卡拉胶）的真实 segment 数据，各类型呈现如下特征：

| semantic_type | 典型内容形态 | 主要挑战 |
|---|---|---|
| `limit` | HTML 表格（感官/理化/微生物指标）、文字描述 | HTML 实体编码（`&lt;`）、表格行转自然语言、脚注保留 |
| `calculation` | LaTeX 块公式 + 变量说明列表 | 纯 LaTeX 对向量化友好度低，需补充用途语义 |
| `procedure` | 含 LaTeX inline 数值的操作步骤，有跨节引用（"A.2.2.1 溶液"） | LaTeX inline 清洁、跨节引用不能猜测替换 |
| `material` | 试剂/仪器列表，含 LaTeX inline 数值 | LaTeX inline 清洁、配制参数保留 |
| `definition` | 术语定义或技术特征描述 | 主语消解、LaTeX inline 清洁 |
| `scope` | 含"本标准"代词的适用范围段落 | 代词消解（"本标准" → 具体标准号） |
| `amendment` | "——"列表格式的修改记录 | 中文标准特有的破折号列表格式 |
| `metadata` | Markdown 标题、发布/实施日期 | 格式清洁即可，无需扩展 |

---

## 设计方案：方案 B（分层扩展强度）

按内容类型的检索价值和原始格式与自然语言的距离，分三档处理：

- **高扩展**：`limit`、`calculation`——原始格式离自然语言最远，需最多处理
- **中扩展**：`procedure`、`material`、`definition`、`scope`——主要解决 LaTeX inline 和代词消解
- **低扩展**：`amendment`、`metadata`——格式清洁为主，不增加语义内容

---

## 各 semantic_type 的 prompt_template 设计

### 高扩展

#### `limit`（技术指标/限量值）

**设计思路**：
- 将 HTML 表格逐行转为独立陈述句，每条指标自包含
- 补充指标类别标注（感官/理化/重金属/微生物）桥接用户的模糊查询（如问"重金属"但原文只写"铅"）
- 类别标注只在表格内容中已有明确依据（如表头行含"理化指标"字样）时才补充，避免 LLM 按常识猜测出错
- 脚注含重要附加条件（如"干燥温度105℃"），必须保留

```
将以下技术指标内容（HTML 表格、Markdown 表格或文字描述）转化为完整陈述文本。

1. 若为表格，将每一个指标行转化为一条独立陈述句，格式为"[指标名称]的要求为[限量值或范围][单位]"；
   示例："铅(Pb)的重金属限量要求为 ≤5.0 mg/kg。"
2. 若表格内容中可明确判断指标类别（如表头行含"感官要求"、"理化指标"、"微生物指标"等字样），
   在每条陈述后补充该类别标注；若无法从原文确定，仅输出指标陈述，不强制添加类别标注。
3. 表格底部的脚注（a、b 等）和"注"包含重要附加条件，须以"备注："开头单独输出，不得省略。
4. 严格保留所有数值、限量符号（≤、<、≥、~、±）、单位，禁止任何改动；HTML 实体编码（&lt; &gt; &amp;）须还原为原始符号。
5. 若内容为文字描述，直接转化为完整独立陈述句，保留所有数值和百分比。
```

#### `calculation`（计算公式）

**设计思路**：
- 纯 LaTeX 文本向量化效果很差，需补充公式用途（"用于计算硫酸酯质量分数"）作为检索入口
- 用途说明只在原文或章节标题中明确出现计算对象时才补充，否则跳过，避免 LLM 从训练记忆中猜测物质名称（同类公式结构在不同 GB 标准中对应不同被测物）
- 块公式（`$$...$$`）保留原样；变量说明中的 LaTeX inline 转为可读符号

```
将以下计算公式及变量说明转化为可检索的混合文本。

1. 若原文或其章节标题中明确给出了计算对象（如"A.3.4 硫酸酯含量的计算"或"硫酸酯的质量分数 w₁，按式(A.1)计算"），
   在最前面用一句话概括计算用途，须包含被测物质名称；若无法从原文确定计算对象，跳过此步，不得猜测。
2. 保留原始 LaTeX 块公式（$$...$$）完整，不做任何修改。
3. 将变量说明中的内联 LaTeX（如 $m_1$、$\mathrm{SO}_4$）转化为可读符号（m₁、SO₄），保留"——"格式的变量说明列表。
4. 严格禁止修改公式中的任何变量名、系数、运算符或数值。
```

---

### 中扩展

#### `procedure`（操作步骤）

**设计思路**：
- LaTeX inline 数值在步骤中大量出现（温度、时间、体积），需转为可读文本
- 跨节引用（"A.2.2.1 溶液"）在 transform 阶段无额外上下文（`ref_context` 仅用于表格引用），告知 LLM 保留引用编号及其描述词，禁止展开猜测；未来版本可通过 ref_context 机制扩展解析

```
将以下操作步骤转化为完整陈述文本。

1. 将内联 LaTeX（$...$）中的数值和单位转化为可读文本（如 $80^{\circ}C$ → 80°C，$15\mathrm{min}$ → 15 min，$7.5\mathrm{g}$ → 7.5 g）。
2. 若步骤中出现章节编号引用（如"A.2.2.1 溶液"、"A.3.3.1 试样"），保留引用编号及其后的简要描述词不变，
   禁止展开或猜测被引用的具体操作内容。
3. 保留所有操作参数（温度、时间、体积、质量、转速等）及其精确数值，禁止任何改动。
4. 步骤编号（如 A.3.3.2）保留在句首作为定位标识。
```

#### `material`（试剂/仪器）

**设计思路**：
- 与 procedure 同样有大量 LaTeX inline 数值
- 通用规定段落（"本标准所用试剂均指分析纯…"）不需要特殊处理，清洁后直接输出

```
将以下试剂、材料或仪器信息转化为完整陈述文本。

1. 将内联 LaTeX（$...$）中的数值和单位转化为可读文本（如 $10\mathrm{g}$ → 10 g，$0.2\mathrm{mol}/\mathrm{L}$ → 0.2 mol/L）。
2. 每条试剂或仪器保留独立陈述，格式为"[名称]：[规格/配制方法]"，保留浓度、纯度、配比等参数。
3. 若内容为通用规定段落（如"本标准所用试剂均指分析纯…"），直接清洁 LaTeX 后输出，不增不减。
4. 禁止修改任何数值、浓度、单位或物质名称。
```

#### `definition`（术语定义）

**设计思路**：
- 主语清晰化是核心，避免 "该物质是指…" 这种孤立存在时语义不完整的情况
- 该类型也会出现技术特征描述（如"卡拉胶有一切多糖具有的典型强宽谱段"）
- 模糊指代替换只针对可在当前文本中确认指代目标的情况，避免替换非指代用途的"该"字（如"该反应具有特异性"）
- 若出现 LaTeX 块公式，转为可读文字描述

```
将以下术语定义或技术特征描述转化为标准化陈述文本。

1. 若包含明确术语定义，输出格式为"[术语] 是指 [定义内容]"，确保主语明确。
2. 若当前文本中出现"本标准中 X 是指"的模式，确保主语"X"在句首明确出现；
   对无法在当前文本中确认指代目标的"上述"、"该"等代词，保留原词不变。
3. 将内联 LaTeX（$...$）转化为可读符号（如 $1000\mathrm{cm}^{-1}$ → 1000 cm⁻¹）；
   若出现块公式（$$...$$），转化为可读文字描述，保留数值和单位不变。
4. 严格保留所有数值、单位、技术参数，禁止任何改动。
```

#### `scope`（适用范围）

**设计思路**：
- "本标准" 是高频代词，直接影响 chunk 的自包含性
- 产品型号中有 LaTeX inline（`$\lambda$`）需转为可读文本
- 标准号只有在当前文本中明确出现时才替换，否则保留"本标准"，禁止从训练记忆中猜测标准号（同一格式的 scope 文本存在于数百个不同 GB 标准中）

```
将以下适用范围文本转化为自包含的陈述文本。

1. 若原文中明确出现具体标准号（如"GB 15044—2016"），将"本标准"替换为该标准号；
   若原文中未出现标准号，保留"本标准"原词，不得猜测或补全。
2. 将内联 LaTeX（$...$）中的物质名称或符号转化为可读文本（如 $\lambda$ → λ）。
3. 保留所有适用对象、原料描述、产品型号等限定信息，不得删减。
4. 输出为一段完整独立的陈述，无需依赖外部上下文即可理解。
```

---

### 低扩展

#### `amendment`（修改记录）

**设计思路**：
- 中文标准的 "——" 列表格式需显式清洁
- 内容精确性最重要，不做任何语义扩展
- 标准号替换规则与 `scope` 保持一致：只在原文中明确出现时才替换

```
将以下标准修改记录转化为完整陈述文本。

1. 将"——"开头的列表项逐条转化为独立陈述句，去除"——"符号。
2. 若原文中明确出现具体标准号（如"GB 15044—2016"），将"本标准"替换为该标准号；
   若原文中未出现标准号，保留"本标准"原词，不得猜测或补全。
3. 严格保留所有修改描述的原文措辞（增加了/删除了/修改了的具体内容），禁止概括或改写。
4. 禁止修改任何指标名称、数值或标准编号。
```

#### `metadata`（标题/文档信息）

**设计思路**：
- 检索价值最低，只做格式清洁
- 覆盖三种情况：Markdown 标题行、发布/实施日期、纯标准号行
- 对纯标准号（如"GB 1886.169—2016"）或纯章节编号标题，直接原样输出，避免 LLM 补充解释性文字

```
将以下章节标题或文档标识信息转化为简洁陈述句。
- 去除 Markdown 标题符号（#、##、###）和多余空格；
- 将发布/实施日期转化为"于[日期]发布，[日期]实施"的格式；
- 若内容为纯标准号（如"GB 1886.169—2016"）或纯章节编号标题，直接原样输出，不添加任何解释或扩展。
```

---

## 通用保真约束

所有 prompt_template 均隐含以下约束（通过类型专属规则落实，而非写一段通用文字）：

- 数值精确性：`≤5.0 mg/kg` 不能变成 `<5.0 mg/kg`
- 单位完整性：`mg/kg`、`CFU/g`、`mol/L` 不能省略或变形
- 化学式保留：`SO₄`、`Pb`、`As` 等不能替换为中文
- LaTeX 处理规则：块公式（`$$...$$`）只在 `calculation` 中保留原样；其他类型的 LaTeX inline 均需转为可读文本
