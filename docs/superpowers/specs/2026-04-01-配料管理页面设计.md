# 配料管理页面设计

## 1. 概述

在 console 项目中新增配料管理页面，用于展示配料列表及配料详情，支持触发配料分析。

## 2. 路由

- 路径：`/ingredients`
- TabNav 新增 tab：`<NavLink end to="/ingredients" className={linkClass}>配料</NavLink>`

## 3. 布局

参考 `ChunksPage`，采用左侧面板 + 右侧详情的分栏布局：

```
┌─────────────────────────────────────────────────────────────┐
│  Header（KB Admin + 统计 + 清空）                            │
├─────────────────────────────────────────────────────────────┤
│  TabNav（Chunks | 上传 | 对话 | 配料）                         │
├──────────────────┬──────────────────────────────────────────┤
│  左侧配料列表      │  右侧配料详情                              │
│  360px           │  flex-1                                  │
│                  │                                          │
│  [搜索框]         │  基本信息卡片                              │
│                  │  ────────────────────                       │
│  - 配料1（选中）  │  is_additive / function_type /           │
│  - 配料2         │  origin_type / limit_usage 等             │
│  - 配料3         │                                          │
│  ...             │  分析结果卡片                              │
│                  │  ────────────────────                       │
│                  │  level / safety_info /                    │
│                  │  alternatives / confidence_score          │
│                  │                                          │
│                  │  [触发分析按钮]                            │
└──────────────────┴──────────────────────────────────────────┘
```

## 4. 左侧配料列表

### 4.1 组件

`IngredientList.tsx`

### 4.2 功能

- 展示配料列表（仅 name 字段）
- 顶部搜索框，支持按 name 过滤
- 点击选中高亮
- 选中后触发右侧详情更新

### 4.3 数据

- API：`GET /api/ingredients?limit=100&offset=0&name={search}`
- 返回：`IngredientsListResponse.items`（数组）

### 4.4 状态

- `ingredients: IngredientInfo[]`
- `loading: boolean`
- `selectedIngredientId: number | null`
- `searchQuery: string`

## 5. 右侧配料详情

### 5.1 组件

`IngredientDetailPanel.tsx`

### 5.2 展示内容

#### 基本信息卡片

| 字段 | 说明 |
|------|------|
| name | 配料名称 |
| is_additive | 是否添加剂 |
| additive_code | 添加剂编码 |
| function_type | 功能类型列表 |
| origin_type | 来源类型 |
| limit_usage | 限用量 |
| cas | CAS 号 |
| safety_info | 安全信息 |

#### 分析结果卡片（若无分析数据，显示空状态）

| 字段 | 说明 |
|------|------|
| level | 风险等级 |
| safety_info | 安全评估 |
| alternatives | 替代建议列表 |
| confidence_score | 置信度 |

### 5.3 触发分析

- 按钮文案："触发分析"
- API：`POST /api/ingredients/{ingredient_id}/analyze`
- 返回：`{ task_id, ingredient_id, status: "queued" }`
- 触发后显示 toast 提示"分析任务已加入队列"
- 成功后刷新详情数据（GET /api/ingredients/{id}）

## 6. API 对接

### 6.1 新增 API 客户端方法

```typescript
// src/api/types.ts
interface IngredientInfo {
  id: number
  name: string
}

// src/api/client.ts
ingredients: {
  list: (params: { name?: string; limit?: number; offset?: number }) =>
    Promise<{ items: IngredientInfo[]; total: number; limit: number; offset: number }>
  get: (id: number) => Promise<IngredientResponse>
  triggerAnalysis: (id: number) => Promise<{ task_id: string; ingredient_id: number; status: string }>
}
```
