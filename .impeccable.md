# Impeccable Design Context — Life Classics

## Design Context

### Users
超市货架前随手拍的消费者。时间紧迫，手机单手持握，注意力分散。核心任务只有一个：**快速判断这个食品能不能买**。不需要阅读长文，需要在 3 秒内看懂结论。

### Brand Personality
**专业 / 可靠 / 精准 / 年轻**

不是医院的冷硬，不是参考 App 的田园绿温情，而是像一台精密仪器——给出的结论让人信服，但界面年轻、有活力、不沉闷。

### Aesthetic Direction
- **新主色**：清透蓝绿（teal/sky 方向），明亮通透，有小清新气质，不冷硬不科技感，适合全年龄段
- **视觉优先，文字最小化**：结果通过颜色、数字、图形传达，而非段落文字
- **信息层级极度聚焦**：一屏只传达一个核心结论，细节折叠在后
- **单一色彩模式为主**：以一套配色为核心，若暗色不影响整体设计语言可选择性添加，但非必须
- **反参考**：不用大量 bullet list，不用插画/吉祥物，不用文字堆砌的分析段落

### Design System Constraints
- OKLCH 色彩体系，CSS 变量语义化（`--background`, `--foreground`, `--card` 等）
- Tailwind CSS 3，UniApp + Vue 3
- 已有组件：Card, Cell, RiskBadge, AITag, BottomBar, Screen, StateView 等可复用
- 风险五档色系（t0–t4，红→橙→黄→绿）保持不变
- 图标库：Remix Icon；字体：DM Sans
- 动效：spring 缓动（`cubic-bezier(0.34, 1.56, 0.64, 1)`），slideUp，floatIn

### Design Principles
1. **3秒原则（设计层面）**：结果呈现后，用户 3 秒内必须能判断食品好坏，无需阅读任何文字。分析处理本身耗时较长，加载态需单独设计（显示分析进度步骤，而非单纯转圈）
2. **视觉代替文字**：分数、颜色环、配料色块传递信息，段落文字折叠隐藏
3. **精准不繁复**：每个元素都有存在理由，装饰服务于信息，绝不为设计而设计
4. **置信感**：排版紧凑有力，数字粗壮，让用户相信这是一个专业工具的结论
5. **年轻感来自节奏**：不是颜色亮，而是动效流畅、间距克制、交互反馈即时
6. **全年龄友好**：触控目标足够大，层级清晰，零学习成本，操作路径不超过2步
