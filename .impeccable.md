# .impeccable.md — 食品安全助手 Design Context

## Design Context

### Users
普通消费者，在超市或家里拿起一款食品时使用。核心场景：「我不确定这个东西能不能吃/应不应该给孩子吃」。使用时间短，需要快速得到答案，不想看大量文字。

### Brand Personality
**友好 · 温暖 · 有趣**
情感目标：用户拍完照等待结果时，感受到的是「有人帮我把关」而不是「我在看一份报告」。结果出来后第一眼得到的是**放心感**，然后才是细节。

### Aesthetic Direction
- **NOT**：绿色吉祥物风格（竞品已做）、医疗/临床感、普通信息流 App
- **IS**：温暖陶土色系（Terracotta + Warm Cream）、大胆排版驱动、留白充足
- 参考气质：像一个懂食品的朋友设计的 App，自信但不冷漠，有个性但不花哨
- 单一颜色模式（无暗色模式）

### Design Principles
1. **放心感优先**：结果页的视觉重心永远是「能不能吃」这一个判断，所有其他信息让位于它
2. **用排版说话**：不依赖插画/吉祥物，用字重、字号、字色的层次建立视觉冲击力
3. **温暖不沉闷**：陶土橙作为点缀色带来能量感，而非医疗蓝/科技灰
4. **诚实直接**：风险标识清晰可辨，不美化也不恐吓，给用户做决策的底气
5. **与竞品拉开距离**：避免绿色主色调、散落卡片、圆润吉祥物等竞品标志性元素
6. **一步完成拍照到判断**：拍照与正在分析应尽量合并在同一页面，以分步骤/折叠的方式降低认知切换成本
7. **渐进式分析反馈**：拍完后用阶段提示/进度条/骨架屏让用户知道“正在做”，而不是长时间空等
8. **结果第一屏先给结论**：当结果需要展示时，首屏优先“一句话结论（能不能吃/风险等级）”，细节与原因后置
9. **分析包含第三步“生成评估报告”**：analysis 页 stepper 保留第 3 步为“生成评估报告”；分析完成后直接跳转 `web/ui/03-result.html`（结果页），不在分析页停留

### Risk Color Scale (重新设计，t0–t4)
- t0 非常安全：`#3D7A5C` 深翠绿，bg `#EEF7F2`
- t1 较安全：`#2E7D7A` 蓝绿，bg `#EAF4F4`
- t2 谨慎：`#B07D1A` 琥珀棕，bg `#FEF5E0`
- t3 存在风险：`#C44A2B` 陶土红，bg `#FDEEE9`
- t4 高风险：`#8B1A1A` 深红，bg `#FDDEDE`

### Tech Stack
UniApp（微信小程序）+ Tailwind CSS，基准宽度 375px

## Result Page Design Rules（结果页 UX 规则）

### Header Collapse Behavior（滚动收起规则）
- **收起形态**：Hero 区域随滚动向上 translate 移出视口，persistent bar 始终固定在顶部
- **禁止**：背景色透明度渐变（半透明色块残留）、用 height collapse 让内容"化掉"
- **收起后 sticky bar**：纯白背景 + 底部 1px 分割线，不带任何彩色背景
- **布局**：返回按钮（左） + 食品名（中，截断） + 风险徽章（紧随食品名右侧，不推到边缘）
- **右上角禁区**：≈ `right: 0` 的 88×32px 区域留给微信胶囊按钮，任何 UI 元素禁止放在此区域

### Bottom Chat Bar（底部操作栏）
- **始终固定**在内容区底部，不随内容滚动
- `position: absolute; bottom: 0` + 滚动区域相应 `padding-bottom` 留出空间
- 渐变遮罩高度适中，不遮挡过多内容

### Status Bar（状态栏）
- 原型中的模拟状态栏（时间、信号）**不需要**，去掉 `.sb` 元素
- 实际小程序中由系统渲染状态栏，顶部安全区由 `padding-top: env(safe-area-inset-top)` 处理

### 微信小程序安全区域
- 右上角：保留约 95px 宽度给胶囊按钮（`right` 方向不放可交互元素）
- 底部：`padding-bottom: env(safe-area-inset-bottom)` 处理底部安全区
